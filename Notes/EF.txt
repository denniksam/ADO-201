Entity Framework Core
Фреймворк для роботи з даними

1. Встановлення
  NuGet:
  - Microsoft.EntityFrameworkCore
      ядро фреймворку
  - Microsoft.EntityFrameworkCore.Tools
      інструменти командного рядку ( PM>   Package Manager Console )
  - Microsoft.EntityFrameworkCore.SqlServer
      драйвери для роботи з MS SQL (якщо працюємо з іншою СУБД - шукаємо
      інший пакет)

2. Режими роботи
  - Data first (DB first) - БД наявна до початку розробки програми, треба
     будувати сутності з таблиць БД
  - Code first - БД немає, створення БД є частиною нашого проєкту
     в залежності від способу створення та внесення змін до БД:
      = Базований на міграціях - більш придатний до змін БД
      = Базований на коді  - простіший для впровадження, але важкий для внесення змін

Вибір: Code first - Migrations
Міграція - автоматично генеровані коди адаптації БД під зміни у програмі

3. Оточення
Центральною частиною EF є контекст даних.
[EFCore]
 - Department (сутність)
 - EfContext : DbContext (контекст даних)
   дані:
   = DbSet<Department> Departments { get; set; }  -- колекція відділів
   конфігурація:
   = override  OnConfiguring(...)  -- інструкції з створення БД. 
     !! на момент створення контексту БД ще не існуе, тому передається 
         рядок підключення до неіснуючої БД

4. Інсталяція
Відкриваємо пакетну консоль 
  (Tools - NuGet Package Manager - Package Manager Console)
  та вводимо
PM> Add-Migration Initial
----- Результат має бути наступним -------------
Build started...
Build succeeded.
To undo this action, use Remove-Migration.
------------------------------------------------
Також у проєкті з'являється папка Migrations, у ній перша міграція (вона ж
відкривається по завершенню команди у консолі)
------------------------------------------------
!! На даному етапі міграція створена, але не застосована
(у БД зміни не внесено). Вводимо у PM
PM> Update-Database
----- Результат має бути наступним -------------
Build started...
Build succeeded.
Applying migration '20230302074756_Initial'.
Done.
-------------------------------------------------

5. Перевірка - підключаємось до БД засобами Студії
Tools - Connect to database -
Provider: MS SqlClient
Server name: (LocalDB)\MSSQLLocalDB
Database name: ef201ado  (з рядка підключення)
-------------------------------------------------

6. Внесення змін (до контексту і БД)
 - створюєму сутність Product
 - додаємо DbSet<Product> Products
 - створюємо міграцію
 - застосовуємо міграцію
