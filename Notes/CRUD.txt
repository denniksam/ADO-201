Зауваження:
 - немає перевірок значень, що підставляються; у всіх даних формально викликається
    ToString()
 - Id - може бути не ініціалізованим, оскільки Guid - ValueType, в ньому завжди є
    якесь значення, без ініціалізації - '000000-000....'
 - Name - не перевіряєть / не екранується (no escaping) - немає перевірок на спецсимволи
    ( ' -> \'   ' -> '' ), наприклад Name = "O'Brian"
     N'{dialog.Product.Name}', -->  N'O'Brian', --> error
 - Price - не перевіряється на <= 0, може бути локалізована (кома замість точки)
    у SQL допускається тільки точка, кома - роздільник параметрів

Видалення даних
!! Реально видаляти дані категорично не рекомендується
 - дані можуть бути зв'язані, це порушить зовнішні ключі
 - каскадні обмеження можуть спричинити видалення зв'язаних даних

 = видалення даних - заповнення окремого поля з інформацією про видалення
  - логічне поле (true/false)
  - дата-час видалення
  - причина видалення (TEXT)
  - журнал-історія видалень та відновлень
